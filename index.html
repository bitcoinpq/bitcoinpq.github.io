BPQ key differences

Bitcoin Post-Quantum is a bit different than Bitcoin.

  • Single address

    Instead of multiple addresses, the best practice is to reuse a single address.

    So in most cases change going to be returned to the same address you issued the transaction from.

    However you are able to provide different change address (just any address in BPQ network - not necessary available in your wallet or yours in general)

  • Limited number of signatures

    In contrast with Bitcoin the address can be used a limited number of times (from a thousand to a million, depending on the chosen height of the Merkle tree).

    Electrum wallet or BPQ QT client will warn you once you have used most of the signatures and when you have number of signatures left enough only to securely transfer the balance to the new BPQ address.

    When creating the address you may select between 3 key levels (10,16 and 20) for 1.000+,65.000+ and 1.000.000+ signatures respectfully.

    Time to generate this keys is different - from seconds to dozens of minutes and also related to computation power your computer has.

Full node setup with Docker

How to start full node with Docker (install Docker) in few clicks:

  1. Clone github repo
    git clone https://github.com/bitcoinpostquantum/docker-bpq.git

  2. Start docker file
    cd docker-bpq
    ./create

  3. Start demon
    ./run_as_interactive
    bpqd -printtoconsole -testnet=1


  4. To get to bpq-cli open new console window and

    sudo docker ps

    you'll see container handle which you need to use in next command:

    sudo docker exec -it ContainerHandle /bin/bash

    bpq-cli -testnet=1 getblockcount


Once you wish to srop container run:

sudo docker stop ContainerHandle

Docker file may be found in this repo
